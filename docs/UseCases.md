## Golden path integration demo

Use the sample to show a realistic flow:

- ingest data from a file source and an HTTP source
- apply config-driven rules
- validate the result
- emit usable output
- inspect a rule-by-rule trace so the decision path is auditable

This is the fastest way to show a client or stakeholder that the repo is about reusable delivery leverage, not disconnected code fragments.
