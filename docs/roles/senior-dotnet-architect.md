# Senior .NET Architect / Library Owner

## Mission

Turn the repo into a real .NET asset with coherent structure, credible APIs, usable examples, and technical proof.

## Full Authority

- repo structure
- project and package layout
- public API shape
- code quality standards
- dependency decisions
- example architecture
- technical cuts and deletions

## Outcomes Owned

- buildable project structure
- coherent grouping of helpers
- removal of duplicated or misleading code
- examples that prove high-value usage
- technical tie-breaking across domains

## First 30 Days

- define the target repo shape
- identify content to delete, merge, rewrite, or promote
- convert draft snippets into a buildable asset
- establish technical acceptance criteria with QA

## Hiring Scorecard

- Can they turn vague code collections into coherent reusable assets?
- Do they cut low-value work instead of preserving it by default?
- Can they make strong interface and package-boundary decisions?
- Do they communicate tradeoffs clearly across code, docs, and product?
- Do they behave like an owner under ambiguity?
