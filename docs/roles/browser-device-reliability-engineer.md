Create real confidence that the public product surface works in production-like browser and device conditions.

- browser automation strategy
- viewport and device coverage
- pre-release smoke validation

- define the browser proof matrix

- add screenshot or visual checkpoints for release review

## Scorecard

- critical paths are green in browser automation

- accessibility-critical interactions are verified before shipping
