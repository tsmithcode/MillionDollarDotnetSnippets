# Accessibility Standards

Million Dollar Dot Net Snippets should feel premium without sacrificing readability, keyboard usability, or assistive-technology clarity.

## Repo-level standards

- every SVG used as a product-facing asset must include a `title` and `desc`
- visual hierarchy must preserve readable contrast
- README navigation and section labels should be scannable in a screen reader
- examples and docs should describe purpose before implementation detail
- decorative polish must never reduce clarity

## Asset standards

- treat SVGs as meaningful product assets, not decorative leftovers
- include descriptive text for hero and mark assets
- preserve strong contrast between text and background
- avoid tiny copy embedded inside visuals

## Content standards

- the first screen should make the audience, product, and next step obvious
- long docs should have clear headings
- critical docs should have one obvious starting point

## Web product standards

- skip links must move focus into the main content region
- external links should disclose that they leave the current product surface
- reduced-motion mode must preserve meaning, not just suppress animation
- desktop and mobile keyboard paths are both part of release proof
- guided flows should explain why a route is recommended and what to do next

## Ownership

- Design / Brand Systems Lead owns visual accessibility
- Accessibility & UX Quality Lead owns repo-wide accessibility quality
- Technical Writer / Developer Experience Editor owns readability and scannability
