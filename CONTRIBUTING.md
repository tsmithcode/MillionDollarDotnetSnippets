# Contributing

MillionDollarDotnetSnippets is run like a product, not a dumping ground.

## Before you open a pull request

- read the [README](./README.md)
- check the [acceptance gates](./docs/acceptance-gates.md)
- review the [operating model](./docs/operating-model.md)
- confirm your change improves proof, usefulness, adoption speed, or positioning honesty

## Contribution standards

- keep public claims aligned with shipped code
- add or update examples when introducing new public helpers
- prefer focused changes with a clear ROI story for the target persona
- do not preserve weak work just because it already exists

## Pull requests should include

- what changed
- why it matters for the highest-ROI persona
- any docs updates required
- any proof added or missing
